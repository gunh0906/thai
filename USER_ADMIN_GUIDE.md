# 한국/태국어 포켓북 사용 방법 및 관리자 메뉴 안내

이 문서는 휴대폰에서 앱을 사용하는 일반 사용자와, 계정과 AI 기능을 관리하는 관리자를 위한 안내서입니다.

기본 접속 주소:

- 라이브 주소: [https://gunh0906.github.io/thai/](https://gunh0906.github.io/thai/)

핵심 특징:

- 로그인 후에만 검색과 AI 번역을 사용할 수 있습니다.
- 한국어 또는 태국어로 검색할 수 있습니다.
- 오른쪽 상단 메뉴는 관리자 기능과 화면 전환에 사용합니다.
- OpenAI API 키는 앱에 넣지 않고 Cloudflare Worker 비밀값에만 저장합니다.

## 1. 일반 사용자 사용 방법

### 1-1. 처음 접속했을 때

1. 앱을 열면 먼저 로그인 화면이 나옵니다.
2. 관리자에게 받은 아이디와 비밀번호를 입력합니다.
3. 처음 받은 임시 비밀번호라면 로그인 후 새 비밀번호로 변경해야 계속 사용할 수 있습니다.

### 1-2. 상단 버튼 설명

- `계정`: 현재 로그인한 아이디를 확인하고 비밀번호를 바꿀 수 있습니다.
- `로그아웃`: 현재 계정에서 바로 로그아웃합니다.
- `🇰🇷 / 🇹🇭`: 앱 표시 언어를 한국어/태국어로 바꿉니다.
- `메뉴`: 관리자만 사용하는 메뉴입니다. 일반 사용자는 보통 건드릴 일이 없습니다.

### 1-3. 검색 방법

검색창에는 아래처럼 바로 쓰고 싶은 말을 그대로 넣으면 됩니다.

- 단어 검색: `화장실`, `수박`, `급여`, `세탁기`
- 짧은 문장 검색: `방 바꿔주세요`, `얼마예요`, `어떻게 가요`
- 태국어 검색: 태국 문자로 검색하면 한국어 뜻과 연결된 결과를 찾습니다.

검색 후 기본 화면:

- `단어`: 핵심 단어 위주 결과로 이동
- `회화`: 바로 써먹는 문장 위주 결과로 이동
- `AI 번역`: 현재 검색창에 적은 문장을 AI가 직접 번역

### 1-4. AI 번역은 언제 쓰면 좋은가

AI 번역은 아래 상황에서 특히 유용합니다.

- 로컬 검색 결과가 약할 때
- 애매한 문장을 자연스럽게 풀고 싶을 때
- 새 문장을 바로 태국어로 만들고 싶을 때

AI 번역 결과 카드에서 보이는 정보:

- 한국어 문장
- 한국어식 태국어 발음
- `태국어 보기` 버튼을 눌렀을 때 태국 문자 원문

### 1-5. 계정 화면에서 할 수 있는 것

계정 버튼을 누르면 아래 기능을 쓸 수 있습니다.

- 현재 로그인 계정 확인
- 현재 비밀번호 입력
- 새 비밀번호 입력
- 비밀번호 변경
- 로그아웃

권장 사항:

- 임시 비밀번호로 받은 계정은 첫 로그인 후 바로 바꾸는 것이 좋습니다.
- 여러 사람이 같은 계정을 같이 쓰지 않는 것이 좋습니다.

## 2. 관리자 메뉴 사용 방법

관리자만 오른쪽 상단 메뉴에서 `관리자 메뉴`를 사용할 수 있습니다.

관리자 화면으로 들어가는 순서:

1. 로그인
2. 오른쪽 상단 `메뉴` 버튼 누르기
3. `관리자 메뉴` 선택
4. 메인 화면이 검색 화면에서 `관리자 작업 공간`으로 바뀝니다

다시 검색 화면으로 돌아가는 방법:

1. 오른쪽 상단 `메뉴`
2. `검색 화면` 선택

중요:

- 관리자 화면에서는 검색창 대신 관리 기능이 메인에 크게 보입니다.
- 작은 메뉴 안에서 억지로 관리하지 않고, 메인 화면에서 사용자 관리와 AI 설정을 다룹니다.

## 3. 관리자 작업 공간: 사용자 관리

`사용자 관리`에서는 새 계정을 만들고 기존 계정을 수정할 수 있습니다.

### 3-1. 새 사용자 만들기

입력 항목:

- `새 아이디`: 예: `worker01`
- `임시 비밀번호`: 8자 이상 권장
- `권한`
  - `일반 사용자`
  - `관리자`
- `AI 사용`
  - 체크 시 AI 번역 사용 허용
- `계정 상태`
  - 체크 시 활성 계정

사용자 추가 후:

- 해당 사용자는 처음 로그인 후 비밀번호를 바꾸게 됩니다.
- AI 사용 체크를 꺼두면 로그인은 가능하지만 AI 번역은 사용할 수 없습니다.

### 3-2. 기존 사용자 관리

사용자 목록 카드에서 확인할 수 있는 항목:

- 현재 권한
- AI 사용 가능 여부
- 계정 활성/비활성 상태
- 비밀번호 재설정

관리 가능한 작업:

- 일반 사용자 ↔ 관리자 변경
- AI 사용 허용/차단
- 계정 활성/비활성
- 새 비밀번호로 재설정

주의:

- 현재 로그인한 관리자 본인 계정은 실수 방지를 위해 일부 권한 변경이 제한됩니다.

## 4. 관리자 작업 공간: AI 연결

`관리자 AI 연결`에서는 앱이 AI 번역을 어떻게 사용할지 정합니다.

설정 항목:

- `사용`
  - 켜면 AI 번역 기능 사용 가능
  - 끄면 전 사용자에게 AI 기능 비활성
- `실행 방식`
  - `수동 버튼`
  - `결과 없을 때 자동`
  - `결과 부족 시 자동`
  - `LLM 전용`
- `프록시 URL`
  - 예: `https://thai-pocketbook-ai.rjsghks87.workers.dev/assist`

### 4-1. AI 모드 설명

`수동 버튼`

- 사용자가 `AI 번역` 버튼을 눌렀을 때만 AI를 호출합니다.
- 토큰 사용량이 가장 적습니다.

`결과 없을 때 자동`

- 로컬 검색 결과가 거의 없을 때만 AI가 자동으로 도와줍니다.
- 비용을 아끼면서 보강하고 싶을 때 가장 무난합니다.

`결과 부족 시 자동`

- 결과가 완전히 없지 않아도 애매하면 AI를 자동으로 호출합니다.
- 비용과 품질의 균형형입니다.

`LLM 전용`

- 일반 검색보다 AI 번역 결과를 더 강하게 사용합니다.
- 품질은 좋을 수 있지만 비용이 가장 큽니다.

관리 권장값:

- 기본 운영: `결과 없을 때 자동`
- 테스트/보수: `수동 버튼`
- 비용보다 품질 우선: `결과 부족 시 자동`

### 4-2. 프록시 URL에 넣어야 하는 값

앱에 넣는 값:

- `https://<worker>.workers.dev/assist`

앱에 넣으면 안 되는 값:

- `sk-...` 로 시작하는 OpenAI API 키
- `https://api.openai.com/v1/...`

중요:

- OpenAI 키는 휴대폰 앱이나 GitHub Pages에 넣지 않습니다.
- 키는 Cloudflare Worker의 secret로만 저장합니다.

## 5. 최초 관리자 계정 준비 방법

이 프로젝트는 더 이상 코드 안에 기본 관리자 계정을 하드코딩하지 않습니다.

최초 관리자 계정은 Worker 쪽 설정으로 준비합니다.

필요한 값:

- `BOOTSTRAP_ADMIN_USERNAME`
- `BOOTSTRAP_ADMIN_PASSWORD`

권장 순서:

1. Worker에 `OPENAI_API_KEY` secret 등록
2. Worker 환경변수에 `BOOTSTRAP_ADMIN_USERNAME` 설정
3. Worker secret에 `BOOTSTRAP_ADMIN_PASSWORD` 등록
4. 배포
5. 해당 계정으로 최초 로그인
6. 관리자 비밀번호를 바로 변경
7. 일반 사용자 계정 추가

참고 파일:

- Worker 안내: [README.md](/D:/Development/thai_phrase_web/workers/openai-assist/README.md)

## 6. 검색 운영 팁

검색 품질을 높이려면 아래처럼 쓰는 것이 좋습니다.

- 단어만 찾을 때: `화장실`, `매표소`, `세탁기`
- 문장 찾을 때: `방 바꿔주세요`, `이거 얼마예요`, `어떻게 가요`
- AI 번역을 쓸 때: 검색창에 문장 전체를 그대로 입력

추천 방식:

- 단어/짧은 회화는 먼저 로컬 검색
- 없거나 애매하면 `AI 번역`
- 특정 장소나 사물을 넣고 싶으면 문장에 같이 적기
  - 예: `공장에 어떻게 가요`
  - 예: `기숙사로 어떻게 가요`

## 7. 문제 해결

### 로그인 관련

`로그인이 안 돼요`

- 아이디와 비밀번호를 다시 확인합니다.
- 관리자가 계정을 비활성화했는지 확인합니다.
- 임시 비밀번호라면 첫 로그인 후 변경이 필요한 상태인지 확인합니다.

`로그인 후 바로 비밀번호 변경하라고 나와요`

- 정상 동작입니다.
- 처음 받은 계정은 새 비밀번호로 바꿔야 계속 사용할 수 있습니다.

### AI 관련

`AI 번역이 안 돼요`

- 해당 계정에 `AI 사용 허용`이 켜져 있는지 확인합니다.
- 관리자 메뉴의 `관리자 AI 연결`에서 `사용`이 켜져 있는지 확인합니다.
- 프록시 URL이 `/assist`로 끝나는지 확인합니다.

`앱에 API 키를 넣는 칸이 안 보여요`

- 정상입니다.
- API 키는 앱에 넣지 않습니다.
- 관리자도 앱에는 프록시 URL만 넣습니다.

### 검색 관련

`원하는 문장이 잘 안 나와요`

- 문장을 좀 더 직접적으로 입력합니다.
- 단어만이 아니라 실제 말할 문장으로 검색합니다.
- 필요하면 `AI 번역` 버튼으로 직접 번역합니다.

## 8. 운영 권장 정책

소규모 운영에서 권장하는 방식:

- 일반 사용자 계정은 사람별로 따로 발급
- 관리자 계정은 소수만 유지
- AI 모드는 `결과 없을 때 자동` 또는 `수동 버튼`
- OpenAI 키는 Worker secret에만 보관
- 임시 비밀번호 계정은 첫 로그인 후 즉시 변경

## 9. 빠른 요약

일반 사용자:

1. 로그인
2. 검색
3. 필요하면 `AI 번역`
4. `태국어 보기`로 현지인에게 보여주기

관리자:

1. 로그인
2. 오른쪽 상단 `메뉴`
3. `관리자 메뉴`
4. 사용자 계정 생성/권한 관리
5. `관리자 AI 연결`에서 AI 모드와 프록시 URL 관리
