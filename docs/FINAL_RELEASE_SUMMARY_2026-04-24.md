# Final Release Summary

작성일: 2026-04-24

## 완료 범위

- Phase 1~5 계획 기준으로 앱 진입점 분리, 검색 런타임 재구성, 로컬 데이터 확장, 성능 개선, AI 번역 UX 개선을 완료했다.
- 일상생활, 회사생활, 공장 내부, 현장, 금형 관련 검색 데이터와 회귀 케이스를 보강했다.
- 최초 로딩 지연을 줄이기 위해 핵심 데이터와 도메인 데이터를 분리하고 필요한 데이터만 지연 로드하도록 정리했다.
- 관리자 사용자 생성, 계정 목록, 삭제, 비밀번호 재설정, 최초 로그인 후 비밀번호 변경 흐름을 보강했다.
- 수동 AI 번역 요청에는 진행 표시를 추가했고, 자동 AI 호출에는 진행 표시가 나오지 않도록 구분했다.
- AI 번역 버튼 라벨을 `AI 번역`으로 통일하고 일반 버튼과 다른 강조 디자인을 적용했다.

## 검증 기록

- 검색 회귀 검증: `node scripts/audit_search.js --regression` 기준 `171 / 171` 통과 기록 확인.
- 태국어/한국어 연결 검증: 라이브 페이지에서 주요 검색 케이스가 의도한 방향으로 연결되는 것을 확인한 기록 보존.
- AI 수동 번역 UX 검증: 수동 요청 중 진행 표시가 보이고, 완료 후 결과 카드로 복귀하는 것을 확인한 기록 보존.
- 버튼 디자인 검증: 일반 버튼과 AI 번역 버튼이 시각적으로 구분되는 것을 확인한 기록 보존.
- 문서 보안 검증: 공개 문서에는 실제 AI 연결 주소, 실제 key 값, 관리자 계정 값, 비밀번호 값을 기록하지 않도록 정리했다.

## 보안 정리 원칙

- 실제 운영 주소와 secret name/value는 공개 문서에 남기지 않는다.
- 최초 관리자 계정 정보와 비밀번호는 비공개 배포 설정과 운영 채널에서만 관리한다.
- 앱 동작에 필요한 런타임 설정은 유지하되, 공개 문서에는 운영 식별자를 쓰지 않는다.
- 로컬 전용 secret 파일은 git 추적 대상이 아니어야 한다.

## 다음 시작점

- 운영 중 새 표현이 발견되면 로컬 데이터와 회귀 케이스를 같이 추가한다.
- AI 번역 품질 이슈는 먼저 로컬 검색 결과와 발음 필드를 확인한 뒤, 필요한 경우 AI 요청 프롬프트를 보정한다.
- 민감 설정 변경은 소스 문서가 아니라 비공개 배포 설정에서 처리한다.
